# Stage5c VSurface Sparse-Guide 机制方法介绍

本文档基于当前代码库实际实现撰写，描述 `stage5c_vsurface` 这条线中已经落地的 `sparse-guide` 机制。示例配置以运行产物 `outputs/stage5c_vsurface/Laboratory_schedule_lr_v1/config.yaml` 为主，对应源码入口是：

- `core/losses/modules.py` 中的 `SparsePointRegularizationLoss`
- `core/losses/builder.py` 中的 loss 构建逻辑
- `train.py` 中的 sparse prior 加载、topology event、学习率调度与 diagnostics 写出

需要先说明一点：本文档写的是“当前实现了什么”，不是“理论上最好应该是什么”。也就是说，文中的行为以代码为准，而不是以早期实验设想为准。

## 1. 机制定位

当前 `stage5c_vsurface` 中的 sparse-guide 不再只是一个简单的“Gaussian center 靠近 sparse point”的辅助项，而是一套由三部分组成的几何引导系统：

1. 稀疏先验支持的连续几何正则  
   核心在 `SparsePointRegularizationLoss`，对 active Gaussians 持续施加 point-to-plane、方向一致性和尺度形状约束。

2. sparse topology 耦合  
   在 `500~3000` 步内，sparse prior 不只进入 loss，还会直接影响 spawn 和 prune 的优先级。

3. 两阶段 schedule  
   `0~3000` 以较强几何塑形为主，`3000~5000` 切换到 tail 阶段，继续保留 sparse 几何约束，但采样、置信度门控、学习率和计算预算都会改变。

一句话概括：这版 sparse-guide 不是“单次初始化”，而是一个贯穿训练前中后期的连续几何控制器。

## 2. 输入数据与预处理

### 2.1 Sparse prior 的来源

训练时使用的 sparse prior 来自场景目录下的 `auxiliaries/colmap_sparse/`，至少包括：

- `points.npy`  
  稀疏点坐标，形状为 `[N, 3]`
- `points_meta.npz`  
  可选 metadata，主要包含 `track_len` 与 `reproj_error`

加载入口在 `train.py` 的 `load_colmap_sparse_points(...)`。

### 2.2 低光照感知的 sparse 点分数

当前实现不仅使用 COLMAP metadata，还会根据训练图像为每个 sparse 点构造两个 low-light 相关分数：

- `brightness_score`
- `gradient_score`

对应实现是 `train.py` 中的 `build_sparse_lowlight_scores(...)`。做法是：

1. 遍历训练视角。
2. 从低光图像或训练图像取出 RGB。
3. 转成灰度图并做稳健归一化。
4. 计算梯度幅值图。
5. 把 sparse 点投影到图像平面。
6. 对可见的 sparse 点累计其亮度和梯度统计。

最后两个分数按如下形式压缩到 `(0,1)`：

`brightness_score = brightness_mean / (brightness_mean + brightness_sigma)`

`gradient_score = gradient_mean / (gradient_mean + gradient_sigma)`

这一步的作用是：在低光照场景里，更亮、更有局部纹理的 sparse 点会得到更高支持权重。

### 2.3 Reliability filter

在真正进入训练前，sparse prior 还会先做一次 reliability filter。对应实现是 `train.py` 中的 `filter_sparse_prior(...)`。

当前示例配置中：

- `RELIABILITY_FILTER_ENABLED: true`
- `FILTER_TRACK_P_LOW: 0.2`
- `FILTER_REPROJ_P_HIGH: 0.8`

实际逻辑是：

1. 对 `track_len` 取低分位阈值  
   保留 `track_len >= quantile(track_len, 0.2)` 的点
2. 对 `reproj_error` 取高分位阈值  
   保留 `reproj_error <= quantile(reproj_error, 0.8)` 的点
3. 默认使用两者交集，即 `track_keep & reproj_keep`
4. 如果交集点数太少，则退化到 `reproj_only`、`track_only` 或 `all`

因此，当前 sparse-guide 不是直接对原始 COLMAP 全量点工作，而是先做一次轻量可靠性筛选。

## 3. Sparse Support Score

进入 `SparsePointRegularizationLoss` 后，每个 sparse 点都会被分配一个 `support_score`，这是后续邻域加权、barycenter 和局部平面估计的基础。

实现函数是 `core/losses/modules.py` 中的 `_get_sparse_support_scores(...)`。

当前支持分数由四项相乘得到：

`support_score = quality_score * density_score * brightness_score * gradient_score`

其中：

### 3.1 Quality score

当 metadata 可用时：

`track_score = log(1 + track_len)`

如果 `QUALITY_TRACK_MODE = log_median_norm`，则再用 track 的中位数做归一化。

`error_score = exp(-reproj_error / error_scale)`

其中 `error_scale` 在当前配置中由重投影误差的中位数给出，因为 `QUALITY_ERROR_SCALE_MODE = median`。

最终：

`quality_score = track_score * error_score`

这表示：

- track 越长越可靠
- reprojection error 越小越可靠

### 3.2 Density score

对每个 sparse 点先估计局部 density radius，再用全局中位半径做归一化：

`density_score = clamp(median_radius / local_radius, density_clamp_min, density_clamp_max)`

因此，局部更致密的 sparse 区域会得到更大权重，但又不会无限放大。

### 3.3 Brightness / gradient score

来自前述 low-light 统计。如果配置里关闭这两项，对应分数直接退化为全 1。

## 4. 连续 sparse 几何正则

### 4.1 只约束 active Gaussians

loss 不是对全部 splats 生效，而是先做一次 active 过滤：

`sigmoid(opacity) > MIN_OPACITY`

当前示例配置里 `MIN_OPACITY = 0.2`。

这样做有两个目的：

1. 不把梯度浪费在完全不参与成像的 Gaussian 上
2. 避免 sparse prior 过度牵引还没真正进入渲染的无效 splat

### 4.2 采样而不是全量监督

即使在 active Gaussians 中，loss 也不会每步约束全部 active splats，而是只采样一批：

- `SAMPLE_POINTS = 4096`

这一步的意义不是弱化 sparse prior，而是控制计算预算，并把它维持为一个“持续但局部”的几何正则项。

### 4.3 邻域构建

对每个被采样的 Gaussian center，loss 不直接约束到单个最近 sparse 点，而是构造一个 sparse 支撑邻域。

实现是 `_build_support_neighborhood(...)`：

1. 对 query point 计算到所有 sparse points 的欧式距离
2. 取最近的 `neighbor_k = max(KNN_K, PLANE_K)` 个点
3. 用
   `knn_support / knn_dist`
   作为未归一化权重
4. 归一化后计算加权 barycenter

因此，当前几何先验的基本支撑单元不是“最近单点”，而是“support-aware 的局部 sparse patch”。

### 4.4 局部平面估计

当前 `MODE = point_to_plane`，所以 loss 会对 sparse 邻域做局部平面拟合。

实现是 `_compute_residual_bundle(...)` 中的这条流程：

1. 取支撑邻域 `knn_points`
2. 以 weighted barycenter 为中心做加权协方差
3. 对协方差做 `torch.linalg.eigh`
4. 最小特征值对应的特征向量作为法向 `normal`
5. `eigen_gap = eigvals[:,1] - eigvals[:,0]`
6. 若 `eigen_gap >= PLANE_MIN_EIGEN_GAP`，则认为该局部平面稳定

对应的置信度定义为：

`plane_confidence = eigen_gap / (eigen_gap + plane_min_eigen_gap + eps)`

因此，当前实现里“stable plane”不是拍脑袋判断，而是由 PCA 特征值结构直接决定的。

### 4.5 基础 point-to-plane 残差

对每个 sampled Gaussian，先定义：

`delta = query_point - barycenter`

法向残差：

`normal_residual = |dot(delta, normal)|`

切向残差：

`tangent_residual = ||delta - dot(delta, normal) * normal||`

基础几何残差：

`geometry_residual = normal_residual + tangent_weight * tangent_residual`

当前配置里：

- `TANGENT_WEIGHT = 0.15`

因此当前 point-to-plane 不是“只看法向距离”，而是“法向主导、切向次之”的复合残差。

### 4.6 不稳定平面的 fallback

在前段 `0~3000`，如果某个 sampled Gaussian 周围的 sparse plane 不稳定，那么 residual 不会直接置零，而是退回 barycenter residual：

`barycenter_residual = ||query_point - barycenter||`

前段的最终 residual 是：

- stable plane 样本：使用 plane residual
- unstable plane 样本：使用 barycenter residual

这就是 diagnostics 中 `point_to_plane_fallback_ratio` 的来源。

## 5. 连续表面方向与尺度约束

当前版相对早期 `stage5b` 的关键升级，是把“spawn 初始化时才用一次的局部表面信息”升级成了训练期连续几何约束。

### 5.1 Orientation alignment

当 `ORIENTATION_ENABLED = true` 时，loss 会持续约束 Gaussian 的局部法向与 sparse plane 法向一致。

实现方法：

1. 把 Gaussian 的四元数转成 rotation matrix
2. 默认取局部第 3 轴作为 Gaussian normal axis
3. 与 sparse plane normal 做对齐

定义：

`orientation_alignment_raw = 1 - |dot(gaussian_normal, sparse_normal)|`

再乘权重：

`orientation_loss = ORIENTATION_WEIGHT * orientation_alignment`

当前配置里：

- `ORIENTATION_WEIGHT = 0.05`

### 5.2 Anisotropic scale target

这一项用于持续约束 Gaussian 的切向展开和法向厚度。

实现函数是 `_build_scale_targets(...)` 和 `_extract_surface_aligned_scales(...)`。

当前流程不是用固定尺度，而是用 sparse 邻域的局部几何估计 target：

1. 以邻域第一个点为 anchor
2. 计算其余邻居相对 anchor 的 offset
3. 在 PCA 的两个切向主轴上计算投影 extent
4. 在法向上计算 thickness
5. 同时用 PCA 的 `sqrt(eigvals)` 提供标准差型约束
6. 二者经过 blend、cap、floor 后生成 target

切向尺度目标大致是：

`target_tangent_scale = clamp(tangent_extent * tangent_scale_ratio, min=tangent_scale_min, max=tangent_scale_cap)`

法向尺度目标大致是：

`target_normal_scale = clamp(normal_extent * normal_scale_ratio, min=target_normal_scale_min, max=target_normal_scale_max)`

当前示例配置里：

- `TANGENT_SCALE_RATIO = 0.75`
- `NORMAL_SCALE_RATIO = 0.24`
- `TARGET_TANGENT_STD_BLEND = 0.45`
- `TARGET_TANGENT_STD_CAP_RATIO = 1.15`
- `TARGET_TANGENT_LOCAL_RADIUS_CAP_RATIO = 0.55`
- `TARGET_TANGENT_STD_FLOOR_RATIO = 0.6`

这说明这版 target 已经不是简单的 `local_radius * ratio`，而是一个“局部 extent + PCA std + 自适应 cap”的混合设计。

### 5.3 正确的法向尺度提取

当前实现不会死板地把 `scale[2]` 当成法向尺度，而是先根据当前 Gaussian rotation 与 sparse normal 的对齐关系，找出真正最接近法向的局部轴，再提取 normal scale。

这是一个很重要的实现细节，因为它避免了“姿态还没完全转正时罚错轴”的问题。

### 5.4 Normal scale penalty

除了 target matching 之外，还保留了一个较弱的形状 penalty：

`normal_scale_penalty = relu(normal_scale - tangent_scale)`

再乘：

`NORMAL_SCALE_WEIGHT = 0.05`

这项的作用是避免 Gaussian 法向厚度明显大于切向展开。

## 6. 最终残差与 robust loss

前段 `0~3000`，最终 residual 由以下几项构成：

1. `point-to-plane` 主残差
2. `orientation_loss`
3. `anisotropic_scale_target_loss`
4. `normal_scale_loss`

再经过一个 Charbonnier 风格的 robust 变换：

`robust_loss = sqrt(residual^2 + robust_scale^2) - robust_scale`

当前配置里：

- `ROBUST_SCALE = 0.05`

最终 sparse loss 取 sampled batch 的均值，再乘当前 step 的 sparse 权重。

## 7. 采样与难例挖掘

### 7.1 前段采样

当前示例配置中：

- `SAMPLING_MODE = hardest_global_mixed`
- `HARD_RATIO = 0.75`
- `DIFFICULTY_SCORE = vsurface_mixed`

但这里有一个必须写清楚的代码细节：

当前全局 hard mining 并不会在全量 active Gaussians 上真正计算完整的 `vsurface_mixed`。  
出于计算效率考虑，`_global_mining_difficulty_mode()` 会把全局扫描时的 difficulty 回退到 `min_sparse_dist`。

也就是说：

1. 全局 hardest 候选筛选时，使用更便宜的距离型难度
2. 对已经 sampled 的 batch，再计算真正的 `vsurface_mixed` diagnostics 与细粒度几何量

这是当前代码里一个明确的工程折中。

### 7.2 `vsurface_mixed` difficulty

对 sampled batch 或局部候选，`vsurface_mixed` 定义为：

`distance_weight * normalized_distance`
`+ orientation_weight * orientation_alignment_raw`
`+ scale_weight * anisotropic_scale_target_raw`
`+ normal_weight * normalized_normal_residual`

当前默认权重：

- `DIFFICULTY_DISTANCE_WEIGHT = 1.0`
- `DIFFICULTY_ORIENTATION_WEIGHT = 0.5`
- `DIFFICULTY_SCALE_WEIGHT = 1.0`
- `DIFFICULTY_NORMAL_WEIGHT = 0.5`

因此，它不是单纯“离 sparse 点远就是 hardest”，而是“距离、方向、尺度、法向残差”的混合难度。

## 8. 两阶段 sparse schedule

### 8.1 时间划分

以 `Laboratory_schedule_lr_v1` 为例：

- `START_STEP = 0`
- `TAIL_START_STEP = 3000`
- `END_STEP = 5000`

代码里 tail 判断是：

`step > TAIL_START_STEP`

所以真正进入 tail 的是 `3001~5000`，不是 `3000~5000`。

### 8.2 Weight schedule

当前使用：

- `WEIGHT_SCHEDULE = plateau_decay`
- `WEIGHT_START_SCALE = 1.0`
- `WEIGHT_END_SCALE = 0.3`
- `TAIL_WEIGHT_HOLD_END_STEP = 4000`
- `WEIGHT_DECAY_END_STEP = 5000`

因此实际权重是：

1. `0~4000`：保持满权重
2. `4000~5000`：线性衰减到 `0.3`

这比早期“从 step 0 就一直衰减”的设计更偏向保住前段几何塑形能力。

## 9. Tail 阶段的 high-confidence 设计

### 9.1 Tail sampling mode

tail 阶段使用：

- `TAIL_SAMPLING_MODE = stable_surface_mixed`
- `TAIL_HARD_RATIO = 0.75`
- `TAIL_MIN_PLANE_CONFIDENCE = 0.35`
- `TAIL_STABLE_SAMPLE_RATIO_FLOOR = 0.5`

其思路是：

1. 不再从全体 active Gaussians 中无差别挖 hardest
2. 优先选择 stable plane 且 plane confidence 高的样本
3. 低置信度稳定样本次之
4. 不足部分再用随机 active 补齐

### 9.2 Tail no-fallback point-to-plane

前段允许 unstable 样本 fallback 到 barycenter residual，但 tail 阶段更严格：

- `TAIL_POINT_TO_PLANE_NO_FALLBACK = true`
- `TAIL_POINT_TO_PLANE_MIN_CONFIDENCE = 0.35`
- `TAIL_POINT_TO_PLANE_WEIGHT_SCALE = 0.2`

也就是说，tail 的 point-to-plane 只对下面这类样本生效：

1. `stable_mask = true`
2. `plane_confidence >= 0.35`

对这些样本，tail residual 会再乘一个 confidence 权重：

`plane_confidence ^ TAIL_POINT_TO_PLANE_CONFIDENCE_POWER`

当前 power 为 1.0，所以就是线性置信度权重。

因此，tail 不是在“所有 active Gaussians 上继续强拉 sparse prior”，而是在“高置信度 stable plane 子集”上继续做几何收束。

### 9.3 Tail difficulty

tail 使用单独的 `stable_surface_mixed` 难度：

`base_score = distance + orientation + scale + normal`

再乘：

`confidence_term = 0.5 + 0.5 * plane_confidence`

最后再乘 stable gate。

当前 tail 权重配置：

- `TAIL_DIFFICULTY_DISTANCE_WEIGHT = 0.75`
- `TAIL_DIFFICULTY_ORIENTATION_WEIGHT = 0.75`
- `TAIL_DIFFICULTY_SCALE_WEIGHT = 1.25`
- `TAIL_DIFFICULTY_NORMAL_WEIGHT = 0.5`
- `TAIL_DIFFICULTY_CONFIDENCE_WEIGHT = 1.0`

说明 tail 比前段更重视尺度失配和高置信度局部平面。

### 9.4 Tail 的计算预算控制

当前代码已经对 tail hard mining 做了专门降本，不再扫描全量 active Gaussians。

配置如下：

- `TAIL_GLOBAL_MINING_REFRESH_INTERVAL = 400`
- `TAIL_CANDIDATE_SUBSET_RATIO = 0.2`
- `TAIL_CANDIDATE_SUBSET_MIN = 32768`
- `TAIL_CANDIDATE_SUBSET_MAX = 65536`

对应行为是：

1. 每 `400` 步才刷新一次 tail hard payload
2. 每次只从 active Gaussians 中随机抽一个候选子集
3. 子集大小为：
   `max(hard_count, hard_count*4, active_count*0.2)`
   再被 `[32768, 65536]` 截断

这一步是一个明确的工程优化，目的只是降低 tail 计算代价，不改变 tail 的总体训练目标。

## 10. Sparse topology 耦合

当前 `stage5c_vsurface` 不是只在 loss 里用 sparse prior，同时还把它接入了 topology event。

### 10.1 事件时序

当前示例配置：

- `SPARSE_TOPOLOGY.START_STEP = 500`
- `PRUNE_START_STEP = 2000`
- `END_STEP = 3000`
- `INTERVAL = 250`

因此拓扑节奏是：

1. `500~1750`：spawn-only
2. `2000~3000`：spawn + prune
3. `3000` 之后 topology 关闭

### 10.2 Surface score

topology 中的 sparse surface score 由 `compute_vsurface_topology_metrics(...)` 给出，本质上复用 sparse loss 的局部几何分析。

定义大致是：

`surface_score = plane_confidence * (orientation_metric + scale_weight * scale_metric + normal_weight * normalized_normal_residual)`

当前配置中：

- `SURFACE_SCORE_NORMAL_WEIGHT = 0.5`
- `SURFACE_SCORE_SCALE_WEIGHT = 2.0`

### 10.3 Spawn

spawn 的核心步骤是：

1. 对每个 sparse 点计算到当前 active Gaussians 的最近距离
2. 若该距离大于自适应 coverage threshold，则判为 coverage hole
3. 以 coverage excess 为基础排序
4. 若启用 surface coupling，则叠加 sparse surface score 与 plane confidence
5. 选出 hole 中优先级最高的一批作为主 spawn

此外，还支持一类 extra surface spawn：

即使某些 sparse 点还没到 coverage hole 条件，只要：

- coverage ratio 已经接近阈值
- plane confidence 足够高
- surface mismatch 足够大

也可以额外补进一小批 surface-driven seeds。

当前配置：

- `MAX_SPAWN_PER_EVENT = 2048`
- `MAX_SURFACE_SPAWN_PER_EVENT = 64`
- `SPAWN_SURFACE_SCORE_WEIGHT = 1.5`
- `SPAWN_SURFACE_ADDITIVE_WEIGHT = 1.0`
- `SPAWN_SURFACE_CONFIDENCE_WEIGHT = 0.75`
- `SPAWN_SURFACE_SLACK_WEIGHT = 0.5`

### 10.4 Spawn 初始化

新的 spawned Gaussians 不会只复制位置，它们的 `quats`、`scales` 和 feature 初始化也都带有 sparse 局部表面信息。

实现是 `train.py` 中的 `prepare_spawn_tensors(...)`。

核心做法：

1. 先在 spawn point 周围估计 sparse 局部平面
2. 构造旋转矩阵 `[tangent1, tangent2, normal]`
3. 转为 quaternion 作为初始姿态
4. 切向尺度和法向尺度由 `local_radius` 乘固定比例得到
5. 若平面不稳定，则退化到基于最近 Gaussian 的 fallback scale
6. 非几何 feature 当前默认使用 `nearest_gaussian_copy`

当前配置中：

- `SEED_FEATURE_INIT = nearest_gaussian_copy`
- `SPARSE_TANGENT_SCALE_RATIO = 0.7`
- `SPARSE_NORMAL_SCALE_RATIO = 0.18`
- `SPAWN_OPACITY = 0.05`

因此，这版 spawn 不是“随机种新点”，而是“在 sparse surface 上定向播种”。

### 10.5 Prune

当前 prune 逻辑是：

1. 候选必须满足 `opacity < PRUNE_OPACITY_THRESH`
2. 同时与 sparse prior 距离足够远
3. 若本次 event 有新 spawn，还要求它离新 spawn 也足够远

基础 prune score 是：

`prune_score = sparse_distance * (1 - opacity)`

若启用 surface coupling，则再乘：

`1 + PRUNE_SURFACE_SCORE_WEIGHT * surface_score`

当前配置中：

- `PRUNE_OPACITY_THRESH = 0.08`
- `PRUNE_DISTANCE_THRESH = 0.12`
- `MAX_PRUNE_RATIO_PER_EVENT = 0.01`
- `PRUNE_SURFACE_SCORE_WEIGHT = 1.0`

这表示 prune 不是单纯删低 opacity 点，而是更倾向于删那些“又低 opacity、又离 sparse surface 远、且局部表面不一致”的点。

## 11. 两阶段学习率调度

当前 `schedule_lr_v1` 版本不是只有 `means` 有 scheduler，而是 `means / scales / quats` 都接入了两阶段指数调度。

实现类是 `train.py` 中的 `TwoPhaseExponentialLRScheduler`。

### 11.1 Means

- `LR_MEANS = 1.2e-4`
- `LR_MEANS_TAIL_TARGET = 1.2e-5`
- `LR_MEANS_FINAL = 1.2e-6`

含义是：

1. 前段 `0~3000` 用较高位置学习率推动几何快速移动
2. tail 开始后先落到 `1.2e-5`
3. 再从 `3000~5000` 指数衰减到 `1.2e-6`

### 11.2 Scales / quats

- `LR_SCALES = 2.5e-3`
- `LR_SCALES_TAIL_TARGET = 3.0e-4`
- `LR_SCALES_FINAL = 2.5e-4`

- `LR_QUATS = 5.0e-4`
- `LR_QUATS_TAIL_TARGET = 6.0e-5`
- `LR_QUATS_FINAL = 5.0e-5`

这体现了当前代码的一个明确设计思想：

- 前段允许 geometry 快速成形
- tail 阶段刻意压低形状和姿态漂移

关键指标包括：

## 14. 当前代码库下的实际经验结论

这一节不是方法设计的一部分，但必须写清楚，否则文档会失真。

当前这版 sparse-guide 机制已经具备以下特征：

1. sparse prior 不再只是弱正则，而是进入了 `spawn / prune / tail / scale / orientation` 的多个环节
2. tail 阶段已经不是简单延长，而是 high-confidence、no-fallback、带预算控制的持续几何约束
3. tangent target 相比更早版本已经明显更自适应，不再总是贴着上限

但从当前实验现象看，这一整套机制更像“强几何正则器”，未必总能带来更好的最终渲染指标。  
尤其在 `Laboratory_schedule_lr_v1` 和 `GearWorks_schedule_lr_v1` 这类 schedule 版本中，虽然 tail 的几何统计更干净，但最终 `5000` 指标并没有优于关闭 sparse-guide 的对应基线。

因此，当前代码库最稳妥的理解不是：

“sparse-guide 越强越好”

而是：

“sparse-guide 是一个可控的几何塑形工具，需要谨慎限定作用时段、作用对象和与 topology 的耦合方式”

## 15. 总结

当前 `stage5c_vsurface` 的 sparse-guide 已经形成了一套完整实现：

1. 从 COLMAP sparse prior 与 metadata 出发
2. 构建质量、密度、低光照感知的 sparse support score
3. 用 point-to-plane、方向一致性和 anisotropic scale target 对 active Gaussians 做连续几何约束
4. 在 `500~3000` 步内通过 sparse topology 直接参与 spawn 和 prune
5. 在 `3000~5000` 切换到 high-confidence tail
6. 通过两阶段学习率与 diagnostics 支持持续调参与机制分析

如果把这版机制用一句话概括，可以写成：

“当前 sparse-guide 是一个由 sparse prior 驱动、同时作用于损失、拓扑和局部几何形状的连续表面约束系统。”
